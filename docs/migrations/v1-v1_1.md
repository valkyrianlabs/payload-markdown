# v1.0.0 → v1.1.0

## Summary

This release simplified code block configuration and removed legacy fields that are no longer used by the renderer.

A small schema reconciliation may be required if you previously customized code block behavior inside stored block config.

## What Changed

Removed fields:

- `prettyCodeBlocks`
- `highlightLines`

Replaced by:

- `enhancedCodeBlocks`

`enhancedCodeBlocks` is now also a deprecated config alias in the newer public API. For current plugin configuration, prefer:

```ts
payloadMarkdown({
  code: {
    enhanced: true,
  },
})
```

## Who Is Affected

You are only affected if:

- you customized Markdown block config, and
- you used `prettyCodeBlocks` or `highlightLines`

If you did not modify code block settings, no action is required.

## Migration Steps

1. Run the app in development mode against a local or backup database.
2. Open a page or post that uses Markdown block config.
3. If no schema warning appears in the console, you are done.
4. If prompted to reconcile fields, replace `prettyCodeBlocks` with `enhancedCodeBlocks`.
5. Repeat if multiple instances are detected.
6. When prompted to remove `highlightLines`, select yes.

## Result

After completing these steps:

- your schema will be aligned with v1.1.0
- code block rendering will behave as expected
- legacy removed fields will be gone from stored block config

## Newer Config Migration

For current plugin configuration, migrate from `config.options` to top-level or collection-level `code`:

```ts
payloadMarkdown({
  code: {
    enhanced: true,
    lineNumbers: true,
    shikiTheme: 'github-dark',
  },
})
```
